# Day 3: Control Flow and Logical Operators

## What I Built
* **Pizza Order System:** A conditional-based program that calculates the final bill for a pizza order based on size, pepperoni add-ons, and extra cheese, handling different price tiers for each.[cite: 1]
* **Treasure Island:** A text-based adventure game that utilizes nested decision trees to lead the player through various paths, resulting in either a "Win" or "Game Over" scenario.

## Key Concepts Learned
* **Conditional Logic:** Mastered the use of `if`, `elif`, and `else` statements to control program flow.
* **Nested Conditionals:** Learned to place `if` statements inside other `if` blocks to handle complex, multi-stage decision-making processes.
* **Logical Operators:** Used `and` / `or` / `not` to combine multiple conditions into a single check.
* **Modulo Operator (`%`):** Understood how to calculate the remainder of a division, essential for identifying patterns or parity.
* **Comparison Operators:** Utilized `<`, `>`, `<=`, `>=`, `!=` and `==` to evaluate the relationship between data points.

## Technical Insights
* **Exclusive vs. Independent Blocks:** Learned that `if/elif/else` acts as an exclusive chain where only the first true condition executes, whereas multiple `if` statements run independently.

## Experience
Day 3 focused on "Character Development" for code—moving from linear scripts to programs that can "think" and make decisions based on user input. I practiced building logic paths from scratch and comparing my implementation against the instructor’s approach, which helped reinforce the syntax and logical structure of Python conditionals.
